# Focus — Design Specification

**Date:** 2026-06-05  
**Domain:** https://focus.etretyakov.ru  
**Status:** Approved for implementation planning

---

## 1. Purpose

Focus — личный веб-сервис для управления текущими активностями. Доступ с любого устройства через браузер. Данные хранятся на сервере и синхронизируются между сессиями.

**Целевой пользователь:** один владелец (автор сервиса). Мультипользовательский режим не требуется на первом этапе.

---

## 2. Функциональные требования

### 2.1 Аутентификация

- Вход по паролю (один аккаунт владельца).
- Сессия сохраняется между визитами (httpOnly cookie).
- Неавторизованный пользователь видит только страницу входа.

### 2.2 Таблица активностей

Главный экран — таблица текущих активностей:

| Колонка    | Описание |
|------------|----------|
| Название   | Текст активности, редактируемый inline |
| Приоритет  | Цветной кружок: 🔴 высокий, 🟡 средний, 🟢 низкий |
| Дедлайн    | Дата (и опционально время); просроченные подсвечиваются |
| Действия   | Развернуть/свернуть, удалить |

Дополнительно:
- Создание новой активности (форма над таблицей или inline-строка).
- Сортировка: по приоритету (по умолчанию), по дедлайну, по дате создания.
- Фильтр: показать все / только незавершённые.
- Drag-and-drop для изменения порядка (nice-to-have, Phase 2 — не в MVP).

### 2.3 Разворачиваемый сниппет (подзадачи)

При клике на строку активности она разворачивается под собой:

- Список подзадач с чекбоксами.
- Добавление новой подзадачи (поле + Enter).
- Отметка выполненной / снятие отметки.
- Удаление подзадачи.
- Прогресс: «3/5 выполнено» в заголовке сниппета.

Активность считается **завершённой**, когда все подзадачи выполнены (или вручную через кнопку «Завершить»). Завершённые активности уходят в архив (скрыты по умолчанию, доступны через фильтр).

### 2.4 Синхронизация между устройствами

- Все изменения сохраняются на сервер немедленно (optimistic UI + API).
- При открытии с другого устройства — актуальное состояние из БД.

## 3. Нефункциональные требования

| Требование | Значение |
|------------|----------|
| Доступность | HTTPS, responsive (mobile + desktop) |
| Производительность | Загрузка главной < 1 с на 4G |
| Безопасность | bcrypt для пароля, CSRF-защита, rate limit на login |
| Деплой | Docker Compose на VPS, домен focus.etretyakov.ru |
| Бэкап | Ежедневный pg_dump (cron на сервере) |

## 4. Архитектура

```
┌─────────────┐     HTTPS      ┌──────────────────┐
│  Browser    │ ◄────────────► │  Next.js App     │
│  (any device)│               │  (App Router)    │
└─────────────┘                └────────┬─────────┘
                                        │
                               ┌────────▼─────────┐
                               │  PostgreSQL 16   │
                               └──────────────────┘
```

**Стек:**

| Слой | Технология | Обоснование |
|------|------------|-------------|
| Frontend | Next.js 15 (App Router) + React 19 | SSR, API routes в одном проекте, простой деплой |
| Стили | Tailwind CSS 4 | Быстрая вёрстка, цветовые приоритеты |
| ORM | Prisma | Типобезопасность, миграции |
| БД | PostgreSQL 16 | Надёжность, бэкапы |
| Auth | iron-session | Лёгкая cookie-сессия для single-user |
| Деплой | Docker Compose + Caddy | Auto-HTTPS для домена |
| Тесты | Vitest + Playwright | Unit + e2e smoke |

## 5. Модель данных

```prisma
model Activity {
  id          String    @id @default(cuid())
  title       String
  priority    Priority  @default(MEDIUM)
  deadline    DateTime?
  sortOrder   Int       @default(0)
  completedAt DateTime?
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
  subtasks    Subtask[]
}

model Subtask {
  id          String    @id @default(cuid())
  activityId  String
  activity    Activity  @relation(fields: [activityId], references: [id], onDelete: Cascade)
  title       String
  completed   Boolean   @default(false)
  sortOrder   Int       @default(0)
  createdAt   DateTime  @default(now())
}

enum Priority {
  HIGH    // red
  MEDIUM  // yellow
  LOW     // green
}
```

Пароль владельца хранится в переменной окружения `OWNER_PASSWORD_HASH` (bcrypt), не в БД.

## 7. API Endpoints

| Method | Path | Описание |
|--------|------|----------|
| POST | `/api/auth/login` | Вход |
| POST | `/api/auth/logout` | Выход |
| GET | `/api/activities` | Список активностей |
| POST | `/api/activities` | Создать активность |
| PATCH | `/api/activities/[id]` | Обновить активность |
| DELETE | `/api/activities/[id]` | Удалить активность |
| POST | `/api/activities/[id]/subtasks` | Добавить подзадачу |
| PATCH | `/api/subtasks/[id]` | Обновить подзадачу |
| DELETE | `/api/subtasks/[id]` | Удалить подзадачу |

Все endpoints (кроме login) требуют авторизованной сессии.

## 9. Scope

### MVP (Phase 1)

- Auth (login/logout)
- CRUD активностей
- CRUD подзадач, чекбоксы
- Таблица с приоритетами и дедлайнами
- Разворачиваемые сниппеты
- Responsive UI
- Docker деплой на focus.etretyakov.ru

### Phase 2 (после MVP)

- Drag-and-drop сортировка
- Архив завершённых активностей с UI
- PWA (offline cache, add to home screen)
- Уведомления о дедлайнах

### Out of scope

- Мультипользовательский режим
- Мобильное нативное приложение
- Интеграции (Telegram, Calendar)
